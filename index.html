(v0.15) Newest Experimental Campus Map Example - Updated 4.17.2012
Changelog:(status: In Progress)
  • Added fitToBounds(): automatically fits/zooms map to bounds of most distant markers
  • Added maxZoom: prevents map from zooming in further than there is map imagery, preventing "sorry we have no imagery here" errors.
  • Replaced fitToBounds with custom function fitToMarkers(), activating it on category open each time, to focus map boundary, zooming and centering to fit the category newly selected.
  • Parking polygon layers (can add & remove now)
  • Parking Layer key loads under the category heading when opened
  • Added Campus Tour Category
  • Added expand/contract indicator arrow to each category
  • On object categories -> required data: name, lattitude, longitude. All other values are programmatically optional to the program, and will be tested before proceeding.
  • Added an animation queue stop() to the category slidetoggle(), to prevent it from building up queued animations.
  • Changed js pattern of array.push(arrayItem) functions to array[array.length] = arrayItem; pattern for a Javascript array performance improvement of 5-50% (according to: this article.)
  • Added Webcams layer that autoupdates webcam images
  • Object images (when available) have percentage-based widths, and scale to fit the proportions of the window (matching the infopane proportions)
  • Fixed IE 7 Legacy issues with drop-down category areas
  • Added default icons to category headings, to further strengthen and clarify the interface link with users
  • Fixed alignment of category indicator arrows in cross-browser fashion

(v0.0) Original Stable Campus Map, totally Data-Driven and flexible - Build Frozen 4.04.2012

Changelog:(status:frozen)
Features:
  • (Parking category disabled)
  • Reads categories and corresponding info strictly from JSON data file.
  • Defaults for each category are set in data file.
  • Infopanes can display name, code, position, info blurb, a marker icon override, and a web link for starters, easily scaling up to more pieces of data in future iterations
  • Markers from each category may be turned on and off independent of each other category
